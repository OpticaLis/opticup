## שלב 1: הרשמה ראשונית (Scenario 1A-S)

**טריגר:** ליד ממלא טופס הרשמה (Monday Forms) → Webhook ל-Make

### סדר בדיקות ומעברים

מיד עם קבלת הטופס, הסינריו בודק לפי **מספר טלפון** בסדר הזה:

```
1. בדיקה ראשונה: האם הטלפון קיים ב-Tier 2 (Master Board)?
   ├─ כן, קיים ב-Tier 2:
   │   ├─ סטטוס "לא מעוניין" / "הסר"
   │   │   → הסטטוס חוזר ל"ממתין"
   │   │   → מקבל הודעת "חזרת למערכת" (טקסט — ממתין לתיעוד)
   │   └─ סטטוס "ממתין" / מסופח לאירוע
   │       → מקבל הודעת "כבר רשום" (SMS + Email)
   │
   └─ לא קיים ב-Tier 2 → בדיקה שנייה:
       2. האם הטלפון קיים ב-Tier 1 (Incoming Leads)?
          ├─ כן, קיים ב-Tier 1:
          │   → מקבל הודעת "כבר רשום" (SMS + Email)
          │
          └─ לא קיים בכלל (ליד חדש לגמרי):
              → נוצר ב-Tier 1 בסטטוס "חדש" (עם כל הפרטים + UTMs)
              → מקבל SMS + Email ברוכים הבאים
              → מופנה לעמוד Success
              → [ידני] עובר ל-Tier 2 אחרי בדיקת פרטים
```

### מה קורה כשליד חדש נוצר ב-Tier 1

**רשומה חדשה ב-Tier 1 (Incoming Leads):**
- סטטוס: **חדש**
- זמן יצירה: timestamp מהרגע שנרשם
- שם מלא, טלפון, אימייל — מהטופס
- UTM fields: Source, Medium, Campaign, Content, Term, Campaign ID — מהטופס
- Interests: SuperSale
- Notes: מתעדכן אוטומטית עם כל הודעה שנשלחת ("הודעת הרשמה נפתחה נשלחה", "קטלוג המחירים נשלח" וכו׳)

**במקביל — סנכרון ל-Affiliates board:**
- אותם פרטים + UTM data נוצרים/מתעדכנים ב-Affiliates board (5089365477)
- ה-Affiliates board מסונכרן עם Facebook Ads scenarios (FB1/FB2/FB3) למדידת איכות קמפיינים

**סטטוסים אפשריים ב-Tier 1:**

| סטטוס | משמעות |
|-------|--------|
| חדש | ליד שנרשם, טרם נבדק |
| רחוק מדי | ליד שגר רחוק מאשקלון — לא רלוונטי לאירוע |
| לא מעוניין | ליד שביטל / לא רוצה |
| לא עונה | ניסו ליצור קשר, לא ענה |
| CALL BACK | צריך להתקשר אליו בחזרה |
| מספר לא תקין | מספר טלפון שגוי |

### בדיקת Tier 1 (ידנית כיום, אוטומטית בעתיד)

לפני המעבר ל-Tier 2, בודקים ב-Tier 1:
- האם הפרטים נכונים (טלפון תקין, שם הגיוני)
- האם הליד רלוונטי (לא רחוק מדי, לא מספר שגוי)

> **🔮 CRM עתידי:** בדיקות אלה יהיו אוטומטיות — ולידציית טלפון, בדיקת מרחק גיאוגרפי

### מעבר מ-Tier 1 ל-Tier 2

- **כיום:** ידני — דניאל מעביר אחרי בדיקת פרטים
- **אין הודעות במעבר הזה** — ההודעות כבר נשלחו ברגע ההרשמה
- סטטוס ב-Tier 2: **ממתין לאירוע**
- ברגע שנפתח אירוע → הליד מקבל הודעה עם קישור הרשמה (שלב 2)

### הקשר בין Tier 2 ל-Tier 3

**עיקרון מפתח:** ליד **לעולם לא נמחק מ-Tier 2**. כשליד נרשם לאירוע ספציפי, **נוצרת רשומה חדשה** ב-Tier 3 (Event Attendees) — הליד לא "עובר" אלא משוכפל. Tier 2 הוא ה-CRM הקבוע, Tier 3 הוא בורד זמני לניהול אירוע בודד. יש חיבור בין הבורדים אבל כל בורד עצמאי. גם ליד שעשה "ביטול דיוור" או "לא מעוניין" — נשאר ב-Tier 2, רק עובר לקבוצה אחרת (Not Interested).

### Tier 2 — Master Board (הבורד הראשי)

**עיקרון מרכזי:** הליד נכנס ל-Tier 2 **פעם אחת ולא יוצא**. זהו ה-CRM המרכזי.

**סטטוסים אפשריים ב-Tier 2:**

| סטטוס | משמעות |
|-------|--------|
| ממתין לאירוע | ליד מאומת, ממתין שייפתח אירוע |
| הוזמן לאירוע | נשלחה לו הודעה שאירוע נפתח + קישור הרשמה |
| אישר הגעה | נרשם לאירוע ספציפי ואישר שיגיע |
| אישר ווידוא הגעה | אישר גם בתזכורת/ווידוא (אישור כפול) |
| לא מעוניין | ביקש לא לקבל יותר הודעות |
| רחוק מדי | גר רחוק מאשקלון — לא רלוונטי |
| ביטל UNSUBSCRIBE | הסיר את עצמו מרשימת התפוצה |

**נתונים מצטברים ב-Tier 2 (לכל ליד):**
- כל ה-UTMs מהרשמה מקורית (Source, Medium, Campaign, Content, Term, Campaign ID)
- **מספרי אירועים שהשתתף בהם** — רשימה מצטברת
- **סכום כולל של רכישות** (₪) — מצטבר מכל האירועים
- Creation Date — זמן יצירת הרשומה ב-Tier 2

**פעולות ידניות אפשריות ב-Tier 2:**
1. **שליחת מייל "תודה שנרשמת" מחדש** — המייל הראשון (Welcome Email)
2. **שליחת קישור לאירוע** — אם יש אירוע פתוח, שולח לליד ספציפי את הקישור להרשמה
3. **רישום ידני לאירוע פתוח** — רישום ליד לאירוע ספציפי ישירות מ-Tier 2

**אוטומציית Meta (Facebook) — שיפור קמפיינים:**
- כשליד **הגיע לאירוע** → הפרטים שלו עוברים ל-Meta כליד ברמה גבוהה (Custom Audience / Conversion)
- כשליד **קנה** → סכום הרכישה עובר ל-Meta כ-Purchase event עם הערך
- **מטרה:** Facebook Ads משתמש בנתונים האלה כדי למצוא לידים דומים (Lookalike) ולשפר את ה-targeting

### Event Status — סטטוסי אירוע (Scenario 5A)

