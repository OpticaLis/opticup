## מצב קיים (Current State)

### Trigger

**Monday Webhook** — `watchBoardItems` on Events Management board (5088674576).
Hook name: "MultiSale Events num." Fires when a **new item is created** on the board.

### Flow Overview

```
New event item created in Events Management board
  ↓
Module 1: Webhook trigger
  ↓
Module 2: Search Events Board for ALL items where Counter = "Last_one"
           Board: 5088674576, column: text_mky8qegk
           Purpose: find the previous "last" event
  ↓
Module 3: AGGREGATOR — MAX(Event ID) across all "Last_one" items
           Extracts the highest Event ID number
  ↓
Module 4: Search for all items where Counter ≠ "Last_one"
           (the newly created event that doesn't have a number yet)
  ↓
Module 7: ROUTER — by event type (Interests dropdown)
  ├── Route 1: SuperSale
  │     Module 5: Update the new event:
  │       - Event ID → MAX + 1
  │       - Counter → "Last_one"
  │       - Form Link → auto-generated SuperSale form URL
  │       - Event Status → "Planning"
  │
  └── Route 2: MultiSale
        Module 10: Update the new event:
          - Event ID → MAX + 1
          - Counter → "Last_one"
          - Form Link → auto-generated MultiSale form URL
          - Event Status → "Planning"
```

### Counter Pattern ("Last_one")

The `Counter` field (`text_mky8qegk`) acts as a marker:
- When a new event is created, it has no counter value
- The scenario finds all events marked "Last_one", takes the MAX Event ID
- Sets the new event's ID to MAX + 1 and marks it "Last_one"
- Previous "Last_one" events keep their marker (not cleared — relies on MAX aggregation)

### Event Status Values

```
Planning → Registration Open → Closed → Event Day → Completed
```

New events always start at "Planning".

## כיוון למערכת חדשה (New System Direction)

### What Changes

1. **Sequential numbering** — uses Supabase RPC with `FOR UPDATE` lock (Iron Rule 11):
   ```sql
   CREATE FUNCTION next_event_number(p_tenant_id UUID)
   RETURNS INTEGER AS $$
     SELECT COALESCE(MAX(event_number), 0) + 1
     FROM crm_events
     WHERE tenant_id = p_tenant_id
     FOR UPDATE;
   $$ LANGUAGE sql;
   ```
2. **Form link** — generated by Optic Up UI, not by Make. Points to internal registration page, not Monday Forms.
3. **No "Counter" hack** — proper auto-increment via RPC replaces the "Last_one" marker pattern.

### What This Scenario Becomes

**Eliminated entirely.** Event creation in Optic Up:
1. User clicks "Create Event" in CRM module
2. RPC `next_event_number()` assigns ID atomically
3. Registration URL auto-generated: `https://app.opticalis.co.il/events/register?id={event_id}`
4. Status defaults to "planning"

No Make scenario needed — this becomes pure Supabase + UI logic.

### CRM Tables Needed

- `crm_events.event_number` — auto-assigned via atomic RPC (not auto-increment, because it's tenant-scoped)
