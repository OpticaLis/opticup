## מצב קיים — מה יש היום

### איך זה עובד

**ההודעות ב-Scenario 8 הן לא אוטומטיות לחלוטין.** יש שני מנגנונים:

#### 1. הודעות לליד בודד — שינוי סטטוס ידני ב-Tier 3
דניאל מעדכן ידנית את הסטטוס של כל ליד ב-Tier 3 (Event Attendees) כדי לגרום
להודעת תזכורת להישלח. זה אומר שכדי לשלוח תזכורת ל-40 נרשמים, צריך לעדכן
40 שורות ידנית.

#### 2. הודעות המוניות לכל הנרשמים — שינוי סטטוס בבורד Events Management
כשדניאל משנה סטטוס מסוים בבורד האירועים (Events Management, Board 5088674576),
נשלחת הודעה לכל הקבוצה בבת אחת. סטטוסים שונים = הודעות שונות.

## כיוון למערכת חדשה — Messaging Hub

### הקונספט

**מסך אחד שמרכז את כל ההודעות של המערכת — אוטומטיות וידניות.**

במקום הודעות מפוזרות על פני 15+ סנריואים ב-Make, כל ההודעות מוגדרות,
נערכות, ונשלחות ממקום אחד. זו אחת ממטרות הפרויקט המרכזיות.

### חלק א׳ — הודעות אוטומטיות (Triggers)

כל שורה בטבלה = כלל אוטומציה:
**"כש-[אירוע X קורה] → שלח [הודעה Y] ב-[ערוץ Z] אחרי [N זמן]"**

#### טריגרים מוגדרים מראש (ברירות מחדל)

| # | טריגר (מתי) | הודעה (מה) | ערוץ (איך) | תזמון (מתי) | הפעיל |
|---|-------------|------------|------------|-------------|-------|
| 1 | ליד נרשם למערכת האירועים | אישור תקנון + ברוכים הבאים | SMS + Email | מיידי | ✅ |

#### יכולות לכל כלל
- **הפעלה/כיבוי** — toggle פשוט
- **עריכת נוסח** — inline, עם תמיכה ב-variables ({{name}}, {{event_date}}, {{coupon}} וכו')
- **שינוי ערוצים** — checkboxes: SMS / WhatsApp / Email
- **שינוי תזמון** — מיידי / אחרי X שעות / X ימים לפני אירוע / שעה ספציפית
- **תנאים** — רק לסטטוס מסוים, רק לשפה מסוימת, רק לאירוע מסוג X
- **שפות** — כל הודעה עם טאב לכל שפה (עברית, רוסית, בעתיד אנגלית)
- **Preview** — תצוגה מקדימה של ההודעה עם נתונים לדוגמה

### חלק ב׳ — הודעות ידניות (Broadcast)

**שליחה חד-פעמית לקבוצה מסוננת — בלחיצה אחת.**

#### פלואו

```
1. בחירת קהל (פילטרים)
   ├── סטטוס: "ממתין לאירוע", "אישר הגעה", "חדש", וכו'
   ├── אירוע: כל האירועים / אירוע ספציפי / אירועים אחרונים
   ├── בורד: Tier 2 (כל הלידים) / Tier 3 (נרשמים לאירוע)
   ├── שפה: עברית / רוסית / הכל
   └── מסנן חופשי: תאריך רישום, מקור, וכו'

2. תצוגת Preview — רשימת הנמענים (שם + טלפון + סטטוס)
   └── מספר נמענים מוצג: "ההודעה תישלח ל-34 אנשים"

3. בחירת ערוץ — SMS / WhatsApp / Email (אחד או יותר)

4. כתיבת הודעה — עורך עם variables
   └── Preview של הודעה לדוגמה עם נתוני ליד אמיתי

5. אישור ושליחה — כפתור "שלח" עם אישור ("בטוח? 34 הודעות")

6. לוג — כל שליחה מתועדת:
   └── מי שלח, מתי, לכמה, באיזה ערוץ, מה הנוסח
```

### לוג הודעות — מבנה

כל הודעה שנשלחת (אוטומטית או ידנית) מתועדת עם:

| שדה | תיאור |
|-----|--------|
| `timestamp` | מתי נשלחה |
| `lead_id` | למי נשלחה |
| `channel` | SMS / WhatsApp / Email |
| `trigger_type` | auto (כלל) / broadcast (ידנית) |
| `trigger_id` | מזהה הכלל (אם auto) |
| `template` | תבנית ההודעה |
| `status` | sent / failed / pending |
| `sent_by` | מי יזם (מערכת / שם המשתמש) |
